Project reorganization - best practice directory structure

## Organization Changes

### New Directory Structure
```
Root (clean entry point):
  ├── README.md           → Project overview
  ├── START_HERE.md       → Student quick start
  │
  ├── deployment/         → All deployment scripts (NEW)
  │   ├── 01-deploy.sh
  │   ├── 02-change-password.sh
  │   ├── 03-setup-ssh-keys.sh
  │   ├── 04-bootstrap-vms.sh
  │   ├── 05-create-cluster.sh
  │   ├── 06-configure.sh
  │   ├── 07-install.sh
  │   ├── 08-verify.sh
  │   ├── cleanup.sh
  │   └── complete-build.sh
  │
  ├── docs/               → All documentation (ORGANIZED)
  │   ├── LAB_GUIDE.md
  │   ├── QUICK_REFERENCE.md
  │   ├── IAM_REQUIREMENTS.md
  │   ├── PROJECT_STATUS.md
  │   ├── FIX-REQUIRED.md
  │   └── instructor/
  │
  ├── scripts/            → Infrastructure automation
  ├── config/             → Team configurations
  ├── lib/                → Shared functions
  ├── archive/            → Legacy content
  ├── state/              → Deployment state
  └── logs/               → Log files
```

### Benefits
✅ Clean root directory (only README + START_HERE)
✅ Numbered deployment scripts (clear sequence)
✅ All docs in docs/ directory
✅ Infrastructure scripts separate from deployment
✅ Easy to navigate for students
✅ Best practice project organization

### Migration Notes

Scripts must now be called with new paths:
```bash
# OLD
./lab-deploy.sh --team 1
./appd-bootstrap-vms.sh --team 1

# NEW
./deployment/01-deploy.sh --team 1
./deployment/04-bootstrap-vms.sh --team 1
```

Documentation now in docs/:
```bash
# OLD
cat LAB_GUIDE.md

# NEW
cat docs/LAB_GUIDE.md
```

All paths tested and verified working.
